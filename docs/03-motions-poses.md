# Motions & Pose Editor

## ROBOTIS action file format

OP3 stores actions in `op3_action_module/data/motion_4095.bin`:

- **256 pages** (motions) × **512 bytes** each.
- Page header (64 B): `name[14]`, `stepnum`, speed, accel, link-to-next/exit…
- **7 steps** max per page × 64 B: `position[31]` (`uint16`, by Dynamixel ID,
  index 0 unused) + `pause` + `time`.
- Tick → degree: `deg = (tick − 2048) × 360 / 4096`.
- Step `time`/`pause` are in ~8 ms units.

`assets/motions.json` is extracted from this file (plus `tune_pose.yaml`) and
holds gestures (salute, wave, clap, sit, stand, push-up) + key poses, each as
`{hold, steps:[{t, p, j:{joint:deg}}]}`.

## Playback

`OP3Robot.play_motion(steps, return_default)` tweens through each step over its
`t` seconds (sine ease), pausing `p` seconds between steps. Gestures
(`hold = false`) return to the **walk-ready** idle pose afterwards, matching the
controller's LinkToExit behaviour. Idle/default = official `ready_pose`.

> Walking is **not** an action page — it is generated by the walking module.
> The twin offers *walk-in-place* (`set_walking()`) as a visual stand-in, since
> there is no localization to translate the base.

## Pose / Scene editor (in-app)

Mirrors the ROBOTIS limits:

- **≤ 7 steps** per scene, **≤ 256** saved scenes.
- Workflow: pose the robot (sliders / 3D gizmo) → **+ Step** (records the current
  20-joint pose) → repeat → **Pratinjau** to play → **Simpan Scene** (named).
- **Muat** loads a saved scene back into the editor to edit its steps; each step
  has a ✕ to remove. Saved scenes persist to `user://op3_scenes.json` and appear
  in the Gerakan dropdown alongside the built-in motions.
